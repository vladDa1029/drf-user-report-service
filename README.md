# DRF User Report Service

Pet-project для изучения Django REST Framework и его возможностей на примере задачи
фильтрации, пагинации и асинхронной генерации отчётов.

Проект реализует API для работы со списком пользователей и генерации HTML-отчётов
на основе тех же фильтров и сортировок, которые используются в API.

---

# Цели проекта

- изучить Django REST Framework
- разобраться с фильтрацией через django-filter
- понять работу ViewSet, pagination и ordering
- реализовать background jobs
- обеспечить совпадение результатов API и экспортируемых отчётов

---

# Основной сценарий

В системе есть список пользователей.

API позволяет:

- получать список пользователей
- применять фильтры
- сортировать результаты
- использовать пагинацию
- создавать отчёт по тем же фильтрам

Отчёт создаётся в фоне, так как для каждого пользователя вычисляется
долгая сводка.

---

# Требования

## Список пользователей

Endpoint:

GET /users/

Поддерживает:

- пагинацию
- фильтрацию
- сортировку

Ответ:

- JSON список пользователей
- метаданные пагинации

---

## Фильтрация

Фильтры могут быть сложными и реализуются через `django-filter`.

Примеры фильтров:

- статус пользователя
- дата регистрации
- роль
- поиск по имени

Фильтрация должна быть **идентична** как для API, так и для генерации отчёта.

---

## Сортировка

Поддерживается сортировка по нескольким полям.

Важно:

порядок пользователей должен быть **детерминированным**, чтобы:

> N-ный пользователь в JSON = N-ный пользователь в отчёте.

---

## Генерация отчёта

Endpoint:

POST /users/export/

Параметры:

те же фильтры и сортировки, что и у списка пользователей.

После запроса:

- создаётся запись отчёта
- запускается background задача
- API возвращает ссылку на отчёт

Пример ответа:

{
    "status": "processing",
    "report_id": "123",
    "url": "/reports/123/"
}

---

## Статус отчёта

Endpoint:

GET /reports/{id}/

Возвращает:

- статус отчёта
- дату создания
- ссылку на файл (если готов)

Статусы:

- pending
- processing
- ready
- failed

---

## Скачивание отчёта

Endpoint:

GET /reports/{id}/download/

Возвращает HTML файл отчёта.

---

# Генерация отчёта

Отчёт содержит:

- список всех пользователей, прошедших фильтр
- для каждого пользователя вычисляется сводка

Сводка имитирует тяжёлое вычисление:

например, функция возвращает случайное число после задержки.

Это нужно для демонстрации background processing.
